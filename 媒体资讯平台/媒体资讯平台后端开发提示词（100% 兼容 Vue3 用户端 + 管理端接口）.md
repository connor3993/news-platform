## 一、项目基本要求

你需要开发一个**媒体资讯平台 Spring Boot 后端**，面向媒体行业的一站式新闻资讯系统，必须覆盖**稿件审核、内容分发、数据看板、认证权限、缓存、定时任务、OSS 文件存储、WebSocket 实时推送**等核心能力。

**核心约束（必须严格遵守）**：

- 必须 100% 兼容 Vue3 管理端和 Vue3 用户端接口
- 所有接口路径、请求参数、响应数据必须与本文档保持一致
- 项目必须包含 Spring Boot 后端、管理端接口、用户端接口三类能力
- 技术实现必须符合项目介绍：Java 21、Spring Boot 4.x、Spring Framework 7.x、MyBatis Plus、Redis、MySQL、JWT、Spring Task、AOP、OSS、WebSocket
- 后端代码必须使用 Spring Boot 4.x 新版生态：`jakarta.*` 命名空间、Java 21 语法与 Maven Compiler `release=21`，不得使用 Spring Boot 2.x、Java 8、`javax.*`、`io.jsonwebtoken:jjwt:0.9.x` 等旧版写法
- 不得加入与媒体资讯平台无关的电商、外卖、支付、购物车等业务
- 重点体现：JWT 多端鉴权、ThreadLocal 用户上下文、Redis 热点缓存、AOP 审计与自动填充、定时统计、OSS 图片托管、WebSocket 审核状态推送

## 二、Maven 多模块架构

项目采用标准 Maven 父子工程结构，共 3 个子模块：

```plaintext
media-news（父工程，统一管理依赖版本）
├── media-common（公共模块）
│   ├── annotation（自定义注解，如 AutoFill、LogRecord）
│   ├── aspect（AOP 切面：审计日志、公共字段自动填充）
│   ├── constant（常量类）
│   ├── context（ThreadLocal 用户上下文）
│   ├── exception（自定义异常）
│   ├── result（统一返回结果）
│   ├── properties（JWT、OSS、Redis 等配置属性）
│   └── utils（JWT、OSS、日期、对象转换工具）
├── media-pojo（数据模型模块）
│   ├── entity（数据库实体）
│   ├── dto（请求参数对象）
│   └── vo（响应视图对象）
└── media-server（核心业务模块）
    ├── config（WebMvc、Redis、WebSocket、MyBatis Plus 配置）
    ├── controller
    │   ├── admin（管理端接口）
    │   ├── user（用户端接口）
    │   └── common（通用接口）
    ├── interceptor（JWT 拦截器）
    ├── mapper（MyBatis Plus Mapper）
    ├── service
    │   └── impl（业务实现）
    ├── task（Spring Task 定时任务）
    └── websocket（审核状态实时推送）
```

### 3.1 Spring Boot 4 / Java 21 编码约束

- 所有 Servlet、Filter、Interceptor、Validation、Multipart 相关导入必须使用 `jakarta.*`，不得使用 `javax.*`
- Maven 父工程统一声明 `java.version=21`，编译插件使用 `<release>21</release>`
- JWT 必须使用 JJWT 0.12.x 新 API，不得使用 0.9.x 的旧版 `Jwts.parser().setSigningKey(...)` 写法
- 依赖版本必须优先选择兼容 Spring Boot 4.x 的稳定版本；如第三方 starter 暂未兼容，需使用其核心包并手动配置
- 启动类必须开启：`@SpringBootApplication`、`@EnableScheduling`，Mapper 扫描使用 `@MapperScan`

## 五、核心公共组件实现

### 5.1 统一返回结果

所有接口必须返回统一格式：

```json
{
  "code": 1,
  "msg": "success",
  "data": {}
}
```

### 5.2 JWT 多端鉴权

- 管理端请求头：`token: {jwt_token}`
- 用户端请求头：`authentication: {jwt_token}`
- JWT payload 必须包含：`userId`、`userType`、`username`
- `userType` 取值：`admin`、`user`

### 5.3 ThreadLocal 用户上下文

实现 `BaseContext`：

- `setCurrentId(Long id)`
- `getCurrentId()`
- `setCurrentType(String type)`
- `getCurrentType()`
- `removeCurrent()`

拦截器解析 JWT 后写入 ThreadLocal，接口完成后必须清理，保证线程安全与用户隔离。

### 5.4 AOP 公共字段自动填充

自定义 `@AutoFill(OperationType.INSERT)` 和 `@AutoFill(OperationType.UPDATE)`：

- 新增时自动填充 `createTime`、`updateTime`、`createUser`、`updateUser`
- 修改时自动填充 `updateTime`、`updateUser`

### 5.5 AOP 审计日志

自定义 `@LogRecord("操作名称")`，记录：

- 操作人 ID、操作端类型
- 请求地址、请求方式、请求参数
- 操作名称、IP、操作时间

### 5.6 全局异常处理

必须统一处理：

- 业务异常
- JWT 过期或非法
- 参数校验异常
- 文件上传异常
- 数据库唯一键冲突

## 八、Redis 缓存策略

必须实现以下缓存：

| 缓存 Key | 内容 | 过期时间 |
| --- | --- | --- |
| `media:category:list` | 启用分类列表 | 24 小时 |
| `media:article:hot` | 热点资讯 Top10 | 10 分钟 |
| `media:article:detail:{id}` | 已发布资讯详情 | 30 分钟 |
| `media:dashboard:today` | 今日看板数据 | 5 分钟 |

写操作必须主动删除相关缓存：

- 分类新增、修改、删除、启用禁用后删除 `media:category:list`
- 稿件发布、下架、审核后删除热点与详情缓存
- 阅读量变化后刷新热点排行

## 九、定时任务实现

使用 Spring Task 实现：

- 每天凌晨 00:10 统计前一日阅读数据，生成 `news_daily_stats`
- 每 10 分钟重新计算热点资讯排行
- 每天凌晨 02:00 清理过期热点缓存

热点分计算建议：

```plaintext
hotScore = viewCount * 1.0 + 最近发布时间权重
```

## 十、接口文档

### 10.1 管理端接口

所有管理端接口路径前缀：`/admin`

所有管理端请求必须携带请求头：`token: {jwt_token}`

#### 管理员登录

- 请求路径：`POST /admin/auth/login`
- 请求参数：

```json
{
  "username": "admin",
  "password": "123456"
}
```

- 响应数据：

```json
{
  "id": 1,
  "username": "admin",
  "name": "管理员",
  "role": "admin",
  "token": "jwt_token"
}
```

#### 退出登录

- 请求路径：`POST /admin/auth/logout`
- 请求参数：无
- 响应数据：无

#### 获取当前管理员信息

- 请求路径：`GET /admin/auth/info`
- 请求参数：无

#### 分类分页查询

- 请求路径：`GET /admin/category/page`
- 请求参数：`page`、`pageSize`、`name`、`status`

#### 新增分类

- 请求路径：`POST /admin/category`
- 请求参数：

```json
{
  "name": "科技",
  "sort": 1,
  "status": 1

```

#### 修改分类

- 请求路径：`PUT /admin/category`

#### 删除分类

- 请求路径：`DELETE /admin/category/{id}`

- 请求路径：`POST /admin/category/status/{status}`

- 请求路径：`GET /admin/article/page`
- 请求参数：`page`、`pageSize`、`title`、`categoryId`、`status`、`beginTime`、`endTime`

#### 查询稿件详情
