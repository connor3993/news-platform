## 一、项目基本要求

你需要开发一个**媒体资讯平台 Vue3 用户端网页版**，用于普通用户浏览新闻资讯、查看热点排行、按分类获取内容、搜索资讯、阅读详情与维护个人资料。

**核心约束（必须严格遵守）**：

- 绝对不能修改后端代码，必须 100% 兼容媒体资讯平台用户端接口
- 前端必须使用 Vue 3，符合项目介绍中“web 管理端与用户端前端都采用 Vue 3”的要求
- 页面功能必须围绕媒体资讯平台：内容分发、热点资讯、分类资讯、阅读统计
- 不得加入购物车、订单、支付、外卖、商品等无关功能
- 所有需要登录的请求必须携带请求头：`authentication: {jwt_token}`
- 分类、资讯列表、资讯详情允许未登录访问；阅读记录、个人资料必须登录

## 二、技术栈要求

- 核心框架：Vue 3（Composition API）
- 构建工具：Vite 4+
- UI 组件库：Element Plus
- 状态管理：Pinia
- 路由：Vue Router 4
- HTTP 客户端：Axios
- 样式：CSS 或 SCSS
- 其他：dayjs、qs

## 三、用户端接口完整文档

所有接口统一返回 `Result<T>` 格式：

```json
{
  "code": 1,
  "msg": "success",
  "data": {}
}
```

所有用户端接口路径前缀：`/user`

所有登录后接口必须携带请求头：

```plaintext
authentication: {jwt_token}
```

### 3.1 认证相关接口

#### 1. 用户注册

- 请求路径：`POST /user/auth/register`
- 请求参数：

```json
{
  "username": "zhangsan",
  "password": "123456",
  "nickname": "张三"
}
```

- 响应数据：无

#### 2. 用户登录

- 请求路径：`POST /user/auth/login`
- 请求参数：

```json
{
  "username": "zhangsan",
  "password": "123456"
}
```

- 响应数据：

```json
{
  "id": 1,
  "username": "zhangsan",
  "nickname": "张三",
  "avatar": "https://xxx.oss-cn.com/avatar.jpg",
  "token": "jwt_token"
}
```

#### 3. 获取当前用户信息

- 请求路径：`GET /user/user/info`
- 请求参数：无

#### 4. 修改用户信息

- 请求路径：`PUT /user/user/info`
- 请求参数：

```json
{
  "nickname": "新昵称",
  "avatar": "https://xxx.oss-cn.com/avatar.jpg"
}
```

### 3.2 分类接口

#### 1. 查询启用分类列表

- 请求路径：`GET /user/category/list`
- 请求参数：无
- 响应数据：

```json
[
  {
    "id": 1,
    "name": "科技",
    "sort": 1
  }
]
```

### 3.3 资讯接口

#### 1. 查询资讯列表

- 请求路径：`GET /user/article/page`
- 请求参数：

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| page | 是 | 页码 |
| pageSize | 是 | 每页条数 |
| categoryId | 否 | 分类 ID |
| keyword | 否 | 标题关键词 |
| sort | 否 | `latest` 最新，`hot` 热度 |

- 响应数据：

```json
{
  "total": 100,
  "records": [
    {
      "id": 1,
      "title": "AI 技术带来媒体生产新变化",
      "summary": "资讯摘要",
      "coverUrl": "https://xxx.oss-cn.com/cover.jpg",
      "categoryId": 1,
      "categoryName": "科技",
      "publishTime": "2026-06-21 10:00:00",
      "viewCount": 2000,
      "hotScore": 2180.5
    }
  ]
}
```

#### 2. 查询资讯详情

- 请求路径：`GET /user/article/{id}`
- 请求参数：无
- 响应数据：

```json
{
  "id": 1,
  "title": "AI 技术带来媒体生产新变化",
  "summary": "资讯摘要",
  "coverUrl": "https://xxx.oss-cn.com/cover.jpg",
  "categoryId": 1,
  "categoryName": "科技",
  "content": "<p>正文内容</p>",
  "publishTime": "2026-06-21 10:00:00",
  "viewCount": 2000
}
```

#### 3. 记录阅读行为

- 请求路径：`POST /user/article/{id}/read`
- 请求参数：无
- 说明：进入详情页后调用，用于阅读数据统计与热度计算。

#### 4. 查询热点资讯排行

- 请求路径：`GET /user/article/hot`
- 请求参数：无

## 五、核心页面功能

### 5.1 登录 / 注册页

- 支持用户名密码登录
- 登录成功后保存 token、用户信息到 Pinia 和 localStorage
- 登录成功后跳转首页
- 注册成功后跳转登录页

### 5.2 首页资讯流

- 顶部展示搜索入口
- 展示分类导航
- 默认加载最新已发布资讯
- 支持无限滚动或分页加载
- 展示标题、摘要、封面、分类、发布时间、阅读量、热度分
- 点击资讯卡片进入详情页

### 5.3 分类资讯页

- 根据分类 ID 查询资讯
- 分类切换时重新拉取列表
- 空数据展示友好占位

### 5.4 搜索页

- 支持按标题关键词搜索
- 搜索请求调用 `/user/article/page`
- 保留最近搜索关键词到 localStorage

### 5.5 资讯详情页

- 展示标题、分类、发布时间、封面、正文 HTML
- 进入页面后调用 `/user/article/{id}/read`
- 正文图片必须自适应容器宽度

### 5.6 热点资讯页

- 调用 `/user/article/hot`
- 按热度分展示 Top10
- 热点数据来自 Redis 缓存，前端不自行计算

### 5.7 个人中心

- 展示头像、昵称、用户名
- 支持修改昵称和头像 URL
- 支持退出登录

## 六、Pinia 状态管理

必须创建以下 Store：

### 6.1 用户 Store

保存：

- token
- userInfo
- isLogin

提供：

- login()
- logout()
- fetchUserInfo()
- updateUserInfo()

### 6.2 分类 Store

保存：

- categoryList

提供：

- fetchCategories()

### 6.3 资讯 Store

保存：

- articleList
- hotList
- currentArticle

提供：

- fetchArticles()
- fetchArticleDetail()
- fetchHotArticles()
- recordRead()

## 七、Axios 封装要求

创建 `src/utils/request.js`：

- baseURL 使用环境变量 `VITE_API_BASE_URL`
- 请求拦截器自动读取用户 token，并设置请求头 `authentication`
- 响应拦截器统一处理 `code !== 1`
- token 失效时清空登录状态并跳转 `/login`

## 九、视觉与交互要求
