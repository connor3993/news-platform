## 一、项目基本要求

你需要开发一个**媒体资讯平台 Vue3 管理端网页版**，用于运营人员和编辑人员完成稿件管理、稿件审核、内容发布、分类维护、数据看板、审计日志查看与文件上传。

**核心约束（必须严格遵守）**：

- 绝对不能修改后端代码，必须 100% 兼容媒体资讯平台管理端接口
- 前端必须使用 Vue 3，符合项目介绍中“web 管理端与用户端前端都采用 Vue 3”的要求
- 页面功能必须围绕媒体资讯平台：稿件审核、内容分发、数据看板、认证权限、审计日志
- 所有管理端请求必须携带请求头：`token: {jwt_token}`
- 必须集成 WebSocket，实时接收稿件审核状态或后台通知
- 不得加入外卖、订单、商品、支付等无关业务

## 二、技术栈要求

- 核心框架：Vue 3（Composition API）
- 构建工具：Vite 4+
- UI 组件库：Element Plus
- 状态管理：Pinia
- 路由：Vue Router 4
- HTTP 客户端：Axios
- 图表库：ECharts 5+
- 富文本编辑器：wangEditor 或同类 Vue3 兼容编辑器
- 样式：CSS 或 SCSS
- 其他：dayjs、qs

## 三、管理端接口完整文档

所有接口统一返回 `Result<T>` 格式：

```json
{
  "code": 1,
  "msg": "success",
  "data": {}
}
```

所有管理端接口路径前缀：`/admin`

所有管理端接口必须携带请求头：

```plaintext
token: {jwt_token}
```

### 3.1 认证接口

#### 1. 管理员登录

- 请求路径：`POST /admin/auth/login`
- 请求参数：

```json
{
  "username": "admin",
  "password": "123456"
}
```

- 响应数据：

```json
{
  "id": 1,
  "username": "admin",
  "name": "管理员",
  "role": "admin",
  "token": "jwt_token"
}
```

#### 2. 退出登录

- 请求路径：`POST /admin/auth/logout`
- 请求参数：无
- 响应数据：无

#### 3. 获取当前管理员信息

- 请求路径：`GET /admin/auth/info`
- 请求参数：无

### 3.2 分类管理接口

#### 1. 分类分页查询

- 请求路径：`GET /admin/category/page`
- 请求参数：`page`、`pageSize`、`name`、`status`

#### 2. 新增分类

- 请求路径：`POST /admin/category`
- 请求参数：

```json
{
  "name": "科技",
  "sort": 1,
  "status": 1
}
```

#### 3. 修改分类

- 请求路径：`PUT /admin/category`
- 请求参数：

```json
{
  "id": 1,
  "name": "科技",
  "sort": 1,
  "status": 1
}
```

#### 4. 删除分类

- 请求路径：`DELETE /admin/category/{id}`

#### 5. 启用 / 禁用分类

- 请求路径：`POST /admin/category/status/{status}`
- 请求参数：`id`

### 3.3 稿件管理接口

#### 1. 稿件分页查询

- 请求路径：`GET /admin/article/page`
- 请求参数：

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| page | 是 | 页码 |
| pageSize | 是 | 每页条数 |
| title | 否 | 标题关键词 |
| categoryId | 否 | 分类 ID |
| status | 否 | 稿件状态 |
| beginTime | 否 | 开始时间 |
| endTime | 否 | 结束时间 |

#### 2. 查询稿件详情

- 请求路径：`GET /admin/article/{id}`

#### 3. 新增稿件草稿

- 请求路径：`POST /admin/article`
- 请求参数：

```json
{
  "title": "AI 技术带来媒体生产新变化",
  "summary": "资讯摘要",
  "coverUrl": "https://xxx.oss-cn.com/cover.jpg",
  "categoryId": 1,
  "content": "<p>正文内容</p>"
}
```

#### 4. 修改稿件

- 请求路径：`PUT /admin/article`

#### 5. 提交审核

- 请求路径：`POST /admin/article/{id}/submit`

#### 6. 审核通过

- 请求路径：`POST /admin/article/{id}/approve`
- 请求参数：

```json
{
  "auditComment": "内容合规，审核通过"
}
```

#### 7. 审核驳回

- 请求路径：`POST /admin/article/{id}/reject`
- 请求参数：

```json
{
  "auditComment": "封面图不符合规范"
}
```

#### 8. 发布稿件

- 请求路径：`POST /admin/article/{id}/publish`

#### 9. 下架稿件

- 请求路径：`POST /admin/article/{id}/offline`

### 3.4 数据看板接口

#### 1. 今日核心数据

- 请求路径：`GET /admin/dashboard/today`
- 响应数据：

```json
{
  "publishCount": 12,
  "viewCount": 30201,
  "auditCount": 18,
  "rejectCount": 2
}
```

#### 2. 阅读趋势

- 请求路径：`GET /admin/statistics/read`
- 请求参数：`begin`、`end`

#### 3. 发布趋势

- 请求路径：`GET /admin/statistics/publish`
- 请求参数：`begin`、`end`

#### 4. 热点资讯 Top10

- 请求路径：`GET /admin/statistics/hot`

### 3.5 审计日志接口

#### 1. 操作日志分页查询

- 请求路径：`GET /admin/log/page`
- 请求参数：`page`、`pageSize`、`operation`、`beginTime`、`endTime`

### 3.6 通用接口

#### 1. 文件上传

- 请求路径：`POST /common/upload`
- 请求类型：`multipart/form-data`
- 请求参数：`file`
- 响应数据：

```json
{
  "url": "https://bucket.oss-cn.com/2026/06/cover.jpg"
}
```

## 四、WebSocket 实时消息推送

管理端必须连接：

```plaintext
ws://{host}/ws/audit/{adminId}
```

要求：

- 登录成功后建立 WebSocket 连接
- 退出登录时主动关闭连接
- 收到审核状态或系统通知时，在页面右上角显示消息提醒
- WebSocket 断开后应自动重连，但最多连续重连 5 次

消息格式：

```json
{
  "type": "ARTICLE_AUDIT",
  "title": "稿件审核状态更新",
  "content": "稿件《AI 技术带来媒体生产新变化》已审核通过",
  "time": "2026-06-21 10:00:00"
}
```

## 六、核心页面功能

### 6.1 登录页

- 支持管理员用户名密码登录
- 登录成功后保存 token、管理员信息到 Pinia 和 localStorage
- 登录成功后跳转数据看板
- 登录后建立 WebSocket 连接

### 6.2 数据看板

- 展示今日发布数、阅读量、审核数、驳回数
- 使用 ECharts 展示阅读趋势
- 使用 ECharts 展示发布趋势
- 展示热点资讯 Top10
- 数据来源必须调用后端接口，不得写死

### 6.3 稿件列表

- 支持按标题、分类、状态、时间范围筛选
- 支持分页查询
- 展示标题、分类、状态、发布时间、阅读量、热度分、操作按钮
- 支持编辑、提交审核、发布、下架

稿件状态映射：

| 状态值 | 展示文案 |
| --- | --- |
| 0 | 草稿 |
| 1 | 待审核 |
| 2 | 审核通过 |
| 3 | 审核驳回 |
| 4 | 已发布 |
| 5 | 已下架 |

### 6.4 新增 / 编辑稿件

- 表单字段：标题、摘要、分类、封面图、正文内容
- 封面图通过 `/common/upload` 上传至 OSS
- 正文使用富文本编辑器
- 支持保存草稿
- 支持保存后提交审核

### 6.5 稿件审核

- 默认只展示待审核稿件
- 支持查看稿件详情
- 支持审核通过
- 支持审核驳回并填写驳回原因
- 审核操作完成后刷新列表，并触发 WebSocket 消息展示

### 6.6 分类管理

- 支持分类分页查询
- 支持新增、修改、删除
- 支持启用 / 禁用
- 分类排序字段必须可编辑

### 6.7 审计日志

- 展示由后端 AOP 记录的操作日志
- 支持按操作名称和时间范围查询
- 展示操作人、操作类型、请求地址、请求方式、IP、操作时间

### 6.8 个人资料

- 展示当前管理员用户名、姓名、角色
- 支持退出登录

### 7.2 WebSocket Store

保存：

- connected

提供：

- connect(adminId)
- disconnect()
- reconnect()
- pushMessage()

### 7.3 字典 Store

保存：

- categoryList
- articleStatusMap

提供：

- fetchCategories()

## 八、Axios 封装要求

创建 `src/utils/request.js`：

- baseURL 使用环境变量 `VITE_API_BASE_URL`
- 请求拦截器自动读取管理端 token，并设置请求头 `token`
- 响应拦截器统一处理 `code !== 1`
- token 失效时清空登录状态、关闭 WebSocket、跳转 `/login`

## 十、视觉与交互要求

- 管理端应保持专业、清晰、高信息密度的后台系统风格

- 审核操作必须二次确认，驳回必须填写原因
